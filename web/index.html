Why one of them lost

No application code chose this. The database did:

CREATE UNIQUE INDEX uq_tickets_flight_seat_active
  ON tickets (flight_id, seat_number)
  WHERE booking_status IN ('confirmed', 'checked_in');

It's partial — cancelled tickets fall outside it, so a released seat can be resold. Capacity is guarded separately by locking the flight row FOR UPDATE, so two callers can't both read "179 of 180 sold" and both proceed.
